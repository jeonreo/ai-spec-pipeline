# /learn — 프롬프트 자동 학습

workspaces/ 최근 실행 결과물을 분석해서 각 스테이지의 SKILL.md 개선 포인트를 제안하고, 승인 시 직접 반영한다.

## 실행 절차

1. `workspaces/local/` 디렉토리에서 최근 3개 세션 폴더를 찾는다 (날짜 기준 최신순)
2. 각 세션의 `out/` 디렉토리에서 스테이지별 결과물을 읽는다
3. 다음 기준으로 각 스테이지 결과물을 평가한다:
   - 출력 형식이 template.md 스키마를 충족하는가
   - 내용이 구체적이고 실행 가능한가 (추상적 표현, "확인 필요" 남발 여부)
   - 비즈니스 정책(policy.md)이 올바르게 반영되었는가
   - 누락된 섹션이나 반복적으로 부족한 항목이 있는가
4. 스테이지별로 개선 가능한 SKILL.md 항목을 목록으로 제시한다
5. 사용자가 승인한 항목만 해당 `prompts/skills/{stage}/SKILL.md`에 반영한다

## 출력 형식

```
## /learn 분석 결과

### 분석 세션
- {세션1 경로} ({날짜})
- {세션2 경로} ({날짜})
- {세션3 경로} ({날짜})

### 스테이지별 개선 제안

#### intake
- [ ] {개선 항목 1}
- [ ] {개선 항목 2}

#### spec
- [ ] {개선 항목 1}

...

### 적용할 항목을 선택해주세요
```

## 주의사항

- policy.md는 직접 수정하지 않는다 (policy-update 스테이지를 통해서만 갱신)
- 결과물이 없는 세션은 건너뛴다
- 제안만 하고, 명시적 승인 없이 SKILL.md를 수정하지 않는다
