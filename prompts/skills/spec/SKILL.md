문제 정의를 분석하여 template.md 구조에 맞는 기능 스펙을 작성한다.

다음 규칙을 반드시 따른다.

- template.md의 섹션 구조를 그대로 유지한다
- 모호한 표현을 사용하지 않는다
- 개발자가 바로 구현할 수 있도록 구체적으로 작성한다
- 설명 문장은 5문장 이내로 간결하게 작성한다
- UI는 컴포넌트와 상태 변화를 명확히 작성한다
- API는 메서드, 경로, 파라미터, 응답 구조를 명시한다
- 예외 처리와 로그 항목은 실제 운영을 고려하여 작성한다
- 불필요한 설명은 제거하고 핵심 정보만 작성한다
