# 비즈니스 정책

아래 정책을 모든 문서 생성 시 최우선으로 반영한다.

## 보안 / 개인정보
- 주민등록번호, 여권번호, 금융정보는 시스템에 저장하지 않는다.
- 개인정보 수집 시 반드시 동의 절차를 명시한다.
- 외부 API 연동 시 인증 방식(OAuth2 / API Key)을 스펙에 반드시 포함한다.

## 서비스 정책
- 모든 기능은 권한(Role) 기반 접근 제어를 기본으로 한다.
- 관리자 기능과 사용자 기능은 명확히 분리한다.
- 변경 이력(수정자, 수정 일시)을 추적해야 하는 항목은 스펙에 명시한다.

## 개발 규칙
- API는 REST 방식을 기본으로 하며, 버전(v1)을 경로에 포함한다.
- 오류 응답 형식: `{ "code": "ERROR_CODE", "message": "설명" }`
- 페이징은 offset/limit 방식을 사용한다.

## Jira 티켓 규칙
- Story Point 기준: 1(2h 이하) / 2(반나절) / 3(하루) / 5(2~3일) / 8(1주)
- 이슈 타입: Epic > Story > Sub-task
- 레이블: `backend`, `frontend`, `infra`, `design` 중 해당 항목 표시

## QA 정책
- 신규 기능은 정상 케이스 + 예외 케이스 + 경계값 케이스를 모두 포함한다.
- 보안 관련 기능(로그인, 권한)은 반드시 부정 케이스를 추가한다.
